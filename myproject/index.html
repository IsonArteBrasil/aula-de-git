Primeiros passos em Git e GitHub

O símbolo $ indica que você deve usar o usuário comum para fazer essa operação.
$ sudo apt install git
O sudo é o comando que dá permissões de super usuário temporariamente; assim é possível fazer a instalação de programas.

O apt do inglês, Advanced Package Tool, em português, Ferramenta de Empacotamento Avançada;
é a ferramenta que nos ajuda na instalação, atualização e desinstalação de programas, entre outras funções.

O install é o comando de instalar, indicando ao apt o que fazer.

Verifique se a instalação foi feita corretamente digitando:
$ git --version Você receberá o retorno da versão instalada. Antes de começar a usá-lo é preciso fazer as configurações iniciais. Assim,
$ git config --global user.name seu_nome_aqui
Insira seu nome no local marcado; seu_nome_aqui.
Em seguida:
$ git config --global user.email seu@email.com
Insira seu email no local marcado; seu@email.com.
Configurando o editor padrão:
$ git config --global core.editor nano
No meu caso, estou usando o nano.

Para verificar as suas configurações:
$ git config --list
Caso queira que as configurações sejam feitas apenas no diretório (não global) indicado,
basta retirar dos comandos a flag --global. Assim:
$ git config user.name seu_nome_aqui
$ git config user.email seu@email.com,
$ git config core.editor nano

Para inicializar o git no diretório desejado (caso ainda não o tenha feito),
vá até o local onde você deseja fazer o controle de versionamento.
$ cd /home/seu_usuário/seu_repositório
O comando cd serve para mudar de diretório/pasta; em seguida digite o caminho até a pasta. e digite $ git init

Ambiente de preparo do Git

Uma das principais funções do Git são os conceitos de Ambiente de Preparação e Commit.
Como você está trabalhando, você pode estar adicionando, editando e removendo arquivos.
Mas Sempre que você atingir um marco ou terminar uma parte do trabalho, você deve adicionar o para um ambiente de preparo.
Arquivos em etapas são arquivos que estão prontos para serem confirmados com o repositório no qual você está trabalhando

Assim, podemos adicionr index.html ao ambiente de preparo.
Exemplo:
git add index.html

Vamos criar também um arquivo README.md que descreve o repositório (recomendado para todos repositórios)

Exemplo:
# hello-world
Hello World repository for Git tutorial
This is an example repository for the Git tutoial on https://www.w3schools.com
This repository is built step by step in the tutorial.

Vamos criar também uma folha de estilos externa básica chamada style.css

E atualize para incluir a folha de estilo:index.html

Agora adicione todos os arquivos no diretório atual ao ambiente de preparo:
Exemplo:
git add --all

Agora todos os 3 arquivos são adicionados ao ambiente de preparo, e estamos prontos para fazer nosso primeiro.commit.
Nota: O comando abreviado para é add --all git add -A

Confirmação do Git

Agora estamos prontos para passar nosso repositório de stage para commit

Devemos adicionar mensagens claras a cada commit, é fácil para si mesmo (e para os outros) para ver o que mudou e quando.
Exemplo:
git commit -m "First release of Hello World!"

Git Commit sem estágio

Às vezes, quando você faz pequenas alterações, usar o ambiente de preparação parece uma perda de tempo.
É possível confirmar alterações diretamente, ignorando o ambiente de preparo.
A opção irá automaticamente organizar cada arquivo alterado, já rastreado.-a

Ao fazer a mudança no repositório, verificamos o status do nosso repositório. Mas, desta vez, usaremos a opção --shortpara ver as mudanças de uma forma mais compacta:
Exemplo:
git status --short
M index.html

Nota: Os sinalizadores de status curtos são:
?? - Arquivos não rastreados
A - Arquivos adicionados ao palco
M - Arquivos modificados
D - Arquivos excluídos
